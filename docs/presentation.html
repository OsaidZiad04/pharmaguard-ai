PharmaGuard AI

Pharmacist-centered AI prescription support.

A safety-first prototype demonstrating OCR intake, pharmacist correction, local RAG, KB governance, safety rules, traceability, and synthetic evaluation.

Prototype only Not clinical validation Pharmacist review required

Problem

Prescription support needs structure, not autonomous advice.

Pressure

Pharmacists review incomplete prescription text and missing context under time constraints.

Risk

Generic AI can appear authoritative even when context is weak or unsupported.

Need

The workflow must preserve pharmacist control, privacy boundaries, and source grounding.

Solution

A command center for draft support and review.

PharmaGuard AI separates OCR intake, correction, extraction, retrieval, safety review, and counseling draft generation so unverified inputs cannot become final medical output.

Draft support only Synthetic data only Patient-facing output disabled

Architecture

Local, modular, evaluation-driven.

Frontend

Next.js pharmacist dashboard with workflow status, correction boundary, source grounding, and evidence page.

Backend

FastAPI services for prescription analysis, OCR intake, local RAG, counseling drafts, and safety findings.

Data

Local Markdown drug profiles, governed registry metadata, synthetic fixtures, and evaluation cases.

OCR Workflow

OCR is assistive and unverified.

  • Images are not stored by default.
  • Mock/synthetic providers remain the safe default.
  • Tesseract is optional, disabled by default, and policy-gated.
  • Only pharmacist-corrected text can move to prescription analysis.

RAG Grounding

Retrieved context stays local and source-visible.

TF-IDF baseline

Local retrieval over Markdown profiles with chunk metadata.

Citations

Retrieved source files, sections, scores, and governance status are displayed.

Insufficient context

Unknown or weak-context cases return insufficient knowledge-base context instead of guessing.

KB Governance

Current knowledge is draft and unvalidated.

  • 15 local draft drug profiles.
  • Registry tracks source status, review status, and clinical-validation status.
  • All current profiles are placeholder educational content.
  • Patient-facing output remains disabled.

Safety Rules

Deterministic prompts, not clinical decisions.

Flags

Missing dose, frequency, duration, route, no medication, unsupported medication, possible identifiers, and governance risk.

Unavailable checks

No real interaction, contraindication, diagnosis, treatment, or patient-specific appropriateness engine.

Demo Flow

Show the workflow boundaries.

  • Open the pharmacist dashboard.
  • Show OCR output as unverified and correction-gated.
  • Analyze corrected text or synthetic prescription text.
  • Review RAG sources, KB governance badges, and safety prompts.
  • Generate draft-only counseling after pharmacist confirmation.

Limitations

Clear boundaries make the prototype credible.

This project is not a medical device, not clinically validated, not a patient-facing advisor, and not a replacement for pharmacists.
  • No real patient data.
  • No trusted clinical source ingestion yet.
  • No production OCR default.
  • No production auth, database, or audit retention system.
